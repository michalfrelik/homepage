Operatory porównania

!=- nie równa się

if ('JavaScript' != 'Java') { //wykonaj kod jeśli }

Porównania:

> - lewa strona większa od prawej
< - prawa strona większa od lewej
>= - lewa strona większa równa od prawej
<= - prawa strona większa równa od lewej
== - obie strony równe
=== - obie strony równe (typeof - sprawdza typ liczba czy string)
!= - obie strony różne
!== - obie strony różne (typeof - sprawdza typ liczba czy string)

Zmienna typeof - definiuje typ danych, czy jest do liczba czy string

const liczba = 7 - liczba
const liczbaString = '7' - string (apostrof ' ' robi różnicę)

liczba == liczbaString - true- JS konwertuje wszystko na string, dlatego równanie jest prawdziwe
liczba === liczbaString - false- typeof sprawdza dokładniej typ danych, rozróżnia string od liczby

Zaleca się używanie potrójnego porównania (typeof) - kod będzie bezpieczniejszy
liczba != liczbaString - false
liczba !== liczbaString - true

Falsy - 6 wartości

- false
- '' ; "" ; `` - pusty string (w JS można używać wszystkich apostrofów - zalecane '')
- 0
- null
- undefined
- NaN - not a number
- false

Truthy - cała reszta

- 'tekst'
- 12
- -4
- {}
- []
- true itd.

AND - && - wszystkie warunki muszą być true

if ((myVariable == 4) && (myVariable == 'tekst')) {...}

OR - || - przynajmniej jeden warunek ma być true

if ((myVariable == 4) || (myVariable != 'tekst')) {...}

NOT - ! - odwraca wartość wyrażenia logicznego

if (!('JavaScript' == 'Java') {...}

!true => false

JS : funkce

Czym jest funkcja?

- na wejściu dostaje dane - argumenty
- dla każdego argumentu zwraca jakąś wartość
- posiada określony wzór

f(x)=x*7
f(x) - argument
x*7 - wynik

function calculate (x) {return x*7;}
function - deklaracja
(x) - parametr
return - słowo kluczowe
x*7 - wynik

function calculate (x){
x=x+3;
console.log(x); retunr x*7:
}

W funkcji możemy zawierać wykonywanie akcji, czasami może być bez argumentu (funkcjie bezargumentowe - może tak być)

Wywołanie funckcji:
- jej nazwa + argument
calculate(2);
- console.log(calculate(2))
- const myCalculation = calculate (4)
console.log(myCalculation)

Funkcja w JS

- Deklarujemy nazwę funckcji (nie zawsze trzeba)
- W środku dokonujemy obliczenia / wykonujemy zadania
- Funkcja przyjmuje argumenty (ale nie musi)
- Funkcja zwraca zawsze coś (nawet jeśli nie mamy return - wtedy zwraca undefined)

Funkcje po co?

- Deklarujemy kod, który się powtarza (DRY - Don't Repeat Yourself) - jeśli mamy coś (np. kod) co się powtarza, to dobrze to umieścić w funkcji
- Zwiększamy czytelność kodu
- Łatwiejsze szukanie błęów
- Odpowiedzialność za wykonywanie zadania wjednym miejscu

Co robić jak nie działa?

Debugging- proces usuwania błędów i ich wyszukiwania

Co robić jak nie działa?

1. Sprawdź czy plik jest zapisany.
2. Sprawdź czy edytujesz właściwy plik
3. Sprawdź czy odświerzyłeś przeglądarkę usuwając tzw. cache (oamięć tymczasowa)
devtools -> Network -> Disable cache (usuwa pliki zapisane tymczasowo, powoduje wczytywanie wszystkich plików na nowo)
4. Sprawdź czy nie masz błędów w edytorze / w devtoolsach
- np. czerwona fala w edytorze
- devtoolsy mówuą gdzie jest błąd, co jest nie tak
- sprawdź DOM można wyłapać błędy w HTML-u
5. Zajrzyj w zakładkę Network - widać czy coś się nie wczytuje
6. "Pogadaj z kaczką" - przeczytaj niby komuś na głos
7. Google it!
- warto skopiować do googla nazwę błędu z konsoli
- najlepiej odpowiedzi szukać do roku wstecz (częste zmiany we frontendzie)
Nie korzystamy ze strony w3schools - dużo przeterminowanych informacji
w3c != w3schools
8. Stack Overflow - strona internetowa
- warto patrzeć na odpowiedzi mające najwięcej plusów
- najlepiej do roku wstecz
