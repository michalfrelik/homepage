Konfiguracja projektu (REPORYTORIUM = PROJEKT)

Reporytorium- opis, mapa wszystkich plików znajdujących się w projekcie
1. Wchodzimy, przez terminal, do katalogu projektu (Homepage).
2. Komenda: git init - założenie reporytorium (git tworzy ukryty katalog .git- przechowucje historię, pliki itp nie wolno go usuwać)

Komendy:

- git status- wyświetla status git-a
- git add .- dodaje wszystkie katalogi, elementy do git-a
- git add index.html- dodaje konkretny katalog do git-a
- gitk- narzędzie do podglądania zmian w naszym repozytorium (projekcie)
- git push- wypchnięcie commitów do GitHub-a - git commit -m "nazwa commitu"- dodaje nowy commit o podanej nazwie

Commit = stan projektu (opis tego co się zmieniło od ostatniego zapisu)

Jak często commitować?
Commitujemy tak często, jak często wprowadzamy zmiany do strony (istotne zmiany). Lepiej częściej niż rzadziej

git lens - rozszerzenie do Visual Studio Code (pozwala na śledzenie zmian)


GitHub != Git

GitHub- platforma pozwalająca na publikowanie kodu z git-a lokalnego na serwer zewnętrzny (GitHubowski), w taki sposób, że inni użytkownicy są w stanie podejrzeć kod (programistyczna wizytówka)

GIT Podsumowanie:

- GIT pozwala na sprawne zarządzanie zmianami w projekcie (
- Działa lokalnie
- Do działania zdalnego potrzebuje serwera (np. GitHub)
- Znajomość podstaw GIT-a jest wymagana niezależnie od technologii
- Konto na GitHubie jest Twoją programistyczną wizytówką, zadbaj aby była ładna i prawdziwa (prawdziwe dane, dużo projektów, cużo commitów)
- Commitujemy jak dokładamy jakąś widoczną zmianę do projektu (lepiej częściej, niż za mało)
- Podstawowy ﬂow: git add . / git commit -m "opis zmiany" / git push

BOX MODEL

Świat stron internetowych jest płaski (prawie płaski)

display: block- blokiem jest paragraf, sekcja, main itp. Pragraf przyjmuje maksymalną szerokość bloku w którym się znajduje (np. praragraf przyjmuje szerokość sekcji w której się znajduje). Wyświetla się na całą szerokość "rodzica"
display: inline- nie rozpycha się jak display: block, nie przenosi tekstu do nowej linii, zajmuje dokładnie tyle ile dany tekst np. strong, b, en, i
display: inline-block

margin: 10px auto; - środkowanie treści (margines górny i dolny - 10 px, boczne marginesy wyśrodkowane)

Jednostki

- 0 - może być bez jednostek
- px - piksele
- % - procenty - brany jest z "rodzica"(np. procent szerokości rodzica)
- vw = viewport width - procent szerokości całego okna (jednosta niezależna od rodzica). Może być stosowane w innych rzeczach np. font-size: 50 vw - czcionka będzie rosła/malała zależnie od szerokośći strony (niepolecane)
- vh = viewport hight

box-sizing
- content box
- border box - zachowuje wymiary boxa jaki zdefiniowaliśmy, możemy zmieniać padding i border a zawsze zostanie zachowany wymiar (element się zmniejszy.zwiększy)

position:
- static - domyślne
- relative - "mapa" dla elementów z innymi własnościami (odniesienie)
- absolute - przykleje box do sekcji/paragrafu itp. z komendą position: relative (przenosi box nad tekst)
- fixed - box podąża za przesuwaniem strony
- sticky - zjeżdżając stroną nasz box przyklei się do krawędzi przeglądarki, wracając zostanie na swoim miejscu (nie jest wspierane przez wszystkie przeglądarki - sprawdzenie na stronie ze slajdów)

top, right, bottom, left - pozycjonowanie boxa (mogą być wartości ujemne)

z-index - box z większym z-indexem będzie wyświetlany wyżej (makszymalnie 9999)

text-decoration: none - kasuje podkreślenia w linkach

float - tekst opływa box/obrazek

overflow: hidden - jeżeli obrazek będzie większy niż sekcja/paragraf/itp to następna sekcja zacznie się nieżej (nie będzie opływała obrazka z sekcji wyższej - bez tego może)

clear: right/left/both - ta sama funkcja co overflow: hidden

BEM = Block Element Modifier

Dlaczego powstał BEM>

- świat stylowania doszedł do wsysokiego stopnia skomplikowania (pecyficzność)
- problem z logicznym nazywaniem klas
- brak ujednoliconego sposobu stylowania
- Importanty , nadpisywanie rzeczy
- brak odwzorowania struktury HTML w CSS
- stylowanie kilkoma klasami, powtarzające się nazwy

BEM jak na dłoni

BEM - przykład z życia

Block - nadrzędny element (dłoń)

Element - część bloku (nie musi być bezpośrednim dzieckiem, ale zawiera się w dłoni)

Modifier - modyfikator - jeśli element występuje w kilu wariantach tworzymy dodatkową klasę

block__element--modifier

BEM jest to sposób pisania

W BEM-ie zalecane jes stosowanie małych liter i myślników zamiast spacni (nie camaleCase)

Podsumowanie

- Szukamy elementów na stronie i dzielimy ją na bloki
- Myślimy o elementach składowych. Robimy analizę i selekcję
- Tworzymy dwie klasy jeśli element ma modyfikator (w modyfikatorze zmieniamy tylko unikalne dla niego wartości, nie przepisujemy rzeczy z klasy bazowej)
