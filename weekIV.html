Konfiguracja projektu (REPORYTORIUM = PROJEKT)

Reporytorium- opis, mapa wszystkich plików znajdujących się w projekcie
1. Wchodzimy, przez terminal, do katalogu projektu (Homepage).
2. Komenda: git init - założenie reporytorium (git tworzy ukryty katalog .git- przechowucje historię, pliki itp nie wolno go usuwać)

Komendy:

- git status- wyświetla status git-a
- git add .- dodaje wszystkie katalogi, elementy do git-a
- git add index.html- dodaje konkretny katalog do git-a
- gitk- narzędzie do podglądania zmian w naszym repozytorium (projekcie)
- git push- wypchnięcie commitów do GitHub-a - git commit -m "nazwa commitu"- dodaje nowy commit o podanej nazwie

Commit = stan projektu (opis tego co się zmieniło od ostatniego zapisu)

Jak często commitować?
Commitujemy tak często, jak często wprowadzamy zmiany do strony (istotne zmiany). Lepiej częściej niż rzadziej

git lens - rozszerzenie do Visual Studio Code (pozwala na śledzenie zmian)


GitHub != Git

GitHub- platforma pozwalająca na publikowanie kodu z git-a lokalnego na serwer zewnętrzny (GitHubowski), w taki sposób, że inni użytkownicy są w stanie podejrzeć kod (programistyczna wizytówka)

GIT Podsumowanie:

- GIT pozwala na sprawne zarządzanie zmianami w projekcie (
- Działa lokalnie
- Do działania zdalnego potrzebuje serwera (np. GitHub)
- Znajomość podstaw GIT-a jest wymagana niezależnie od technologii
- Konto na GitHubie jest Twoją programistyczną wizytówką, zadbaj aby była ładna i prawdziwa (prawdziwe dane, dużo projektów, cużo commitów)
- Commitujemy jak dokładamy jakąś widoczną zmianę do projektu (lepiej częściej, niż za mało)
- Podstawowy ﬂow: git add . / git commit -m "opis zmiany" / git push

BOX MODEL

Świat stron internetowych jest płaski (prawie płaski)

display: block- blokiem jest paragraf, sekcja, main itp. Pragraf przyjmuje maksymalną szerokość bloku w którym się znajduje (np. praragraf przyjmuje szerokość sekcji w której się znajduje). Wyświetla się na całą szerokość "rodzica"
display: inline- nie rozpycha się jak display: block, nie przenosi tekstu do nowej linii, zajmuje dokładnie tyle ile dany tekst np. strong, b, en, i
display: inline-block
